Problem Solving: Getting unblocked

19 September 2023

  1. A time I was blocked on a simple problem:

    Incorrect assumptions about array.prototype.map() and array.prototype.forEach()

    What was the problem?

    The forEach() method wasn't doing what I thought I knew it did; I had given an example of map() converting an array of values into an array of HTML list items, and then tried to do the same example but with forEach() thinking it mutates the array, whereas map does not.

    What problem-solving techniques did you use?

    I tried a few techniques to work out what was going wrong, firstly double checking my code and trying a simple example, and console.logging to double check what was happening. That told me that it wasn't doing what i thought it should do, so I checked MDN:Array.prototype.forEach() and MDN:Array.prototype.map() , I also googled and found some stack overflow posts discussing the differences between map and forEach, and also went back to look at some code I had previously written where I knew I had used forEach().

    How did you feel throughout the process?

    I had been using map and forEach with no problems in the past but somehow I didn't fully understand them, so this was confusing and surprising.

    What did you learn?

    I got a better understanding of how these functions work, but also I realised that you can't always rely on things you think you know, I had made some incorrect assumptions and when trying to explain one of these I found that it was incorrect.

  2. A time I elegantly solved a problem:

    What was the problem?

    Not sure if this was an elegant solution but I'm happy with the final result... I wanted to make my blog list on the front page dynamic, so I could filter and group the posts different ways

    What problem-solving techniques did you use?

    I had an idea of how I could do it, so first I broke the problem down and wrote some high level pseudo code, and worked on each part step by step. There was a lot of console logging and using simple data which I built up slowly to what was actually required (i.e. the full metadata for the posts including post name, url, section name, sprint number). There may have a been a bit of googling, and probably a lot of talking myself through what it was doing and what I needed it to do (i.e. the rubber ducky method).

    How did you feel throughout the process?

    It took me quite a while to get right, starting with the logic then the styling, but I was making good progress throughout and it felt good to be using my JS skills to solve a problem, knowing that I had even more problem solving tools available if I needed them.

    What did you learn?

    I solidified my understanding of using JS to manipulate the DOM, JSON, and also some HTML/CSS, to solve quite a problem that had a few elements to it. And reaffirmed that I enjoy problem solving and the rewarding feeling of getting things done.

  3. Reflecting on the problem solving techniques:

    I'm confident with a wide array of problem-solving techniques and processes, ranging from effective pseudocode planning to reading documentation and online resources to seeking guidance from peers and coaches. This confidence allows me to approach challenges with assurance and adaptability, knowing that I can leverage these tools effectively to find solutions and continuously improve my approach.
