Git Tips: Undoing and Amending Commits

Git, the powerful version control system, offers several useful commands to manage your commits effectively. In this post, we'll explore two essential operations: undoing the last commit and amending the last commit.

Undo the Last Commit

Mistakes happen, and sometimes we need to undo the last commit. Git provides multiple options based on what you want to achieve.

1. Undo and Leave Changes Staged

    
// Resets the HEAD only
git reset --soft HEAD~

2. Undo and Un-stage Changes

    
// Resets the HEAD and index
git reset --mixed HEAD~   // AKA: git reset HEAD~

3. Undo and Discard Changes

    
// Resets the HEAD, index, and working directory
git reset --hard HEAD~

Amending the Last Commit

Sometimes, you may want to add new changes and redo the last commit, either to edit the commit message or to include additional modifications. 

    
// add new changes and redo commit
git commit -a --amend            // to edit the commit message, or
git commit -a --amend --no-edit  // keep the same commit message

This is equivalent to the following steps:

  1. Soft reset the HEAD to the previous commit.
  2. Stage the changes you want to include in the commit.
  3. Commit the changes again.
